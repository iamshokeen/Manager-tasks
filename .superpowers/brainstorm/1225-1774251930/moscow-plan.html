Lohono CMD Redesign Plan

MoSCoW prioritization for the full upgrade — UI/UX overhaul, drag-drop interactions, mobile-first fixes, and Notion-style features. Ordered by impact and feasibility.

M — Must Have
Core experience fixes · Ship these first
Broken or critically low-value without these
📱
Mobile-first responsive layout
Replace fixed 220px sidebar with a bottom navigation bar on mobile. Full-width content on phones. Sidebar collapses to a slide-in drawer on tablets. Currently the app is completely unusable on phone.

Visual identity upgrade — Hero pages
Every page gets a proper hero header with gradient, quick-stats, and context. Replace the plain text headings with structured page headers that include breadcrumbs, actions, and KPI callouts. Dashboard gets a full hero section.
UI refreshHero sections

Drag-and-drop Kanban board
Tasks page gets real drag-drop using dnd-kit. Drag cards between columns (To Do → In Progress → Review → Done). Visual drop targets, drag handles, smooth animations. Works on desktop and tablet touch.

W — Won't Have (now)
Out of scope · Revisit later
Either too complex, changes the product's nature, or low ROI for one user
👥
Multi-user collaboration (real-time)
Live cursors, simultaneous editing, presence indicators. This is a single-user tool for Saksham — not a team product. Real-time sync adds enormous complexity for zero current benefit.
Single-user app
📧
Full Gmail integration
Reading, sending, and threading emails inside the app. Gmail MCP already exists but full in-app email client is scope creep. Keep the existing email report sending, nothing more.
Out of scope
💳
Multi-tenant / SaaS billing
This is a personal tool, not a product. No user accounts, subscriptions, or org-level access control needed.
Not a SaaS
📦
Native iOS/Android apps
PWA already gives a native-like experience on mobile. Building separate native apps is a 10x effort increase for minimal gain over an installable PWA.
PWA covers this
