Lohono Command Center Design Spec

A personal management tool for Saksham Shokeen — Head of Revenue Strategy, Lohono Stays. 5 direct reports, 6 departments, FY27 KPI tracking, team management, and a Notion-style workspace. Single user. Next.js + Tailwind. This document is the complete design brief for every screen.
🎨 Design System
Global tokens, typography, spacing, and components used across all screens.

🌈 All 20 Colour Themes
User can switch between any of these at any time. Each theme defines: background, surface, border, accent, text colors. Status colors (red/amber/green/blue for priority) remain constant across all themes.
Design rule: Every theme uses the same component structure and layout. Only colors change. The accent color is used for: active nav items, primary buttons, progress fills, active tab indicators, selected states, focus rings, and links.

Grid of 4 StatCards: Open Tasks (blue), Overdue (red — red number if >0), Due Today (accent/gold), Completed This Year (green). Each card: label (uppercase tiny) + large number + sub-caption + trend line ("↑ 3 added today"). Hover: card lifts 2px. Click: navigates to Tasks filtered to that state.

Main Content Grid

Component
Left column (65%) of a 2-column grid. Shows a mini Kanban: To Do / In Progress / Done columns (Review omitted for space). 2–3 task chips per column visible. Component
Left column. Shows active team members as compact cards (avatar + name + role + task count). Task count turns red if >5. Clicking a member card opens their profile. "View all →" link. Shows max 5 members, sorted by task count descending.

Component
Center column. Notion-style list of recently edited pages. Each row: page emoji icon + title + either a tag badge (Strategy, SOPs) or relative timestamp ("2 days ago"). Hover highlights row. Click opens the page editor. "+ New page" action in header. Shows max 5 items.

Component

✅ Tasks Must Have
The primary work surface. All tasks across all team members and projects. Switchable between Kanban, Table, Calendar, and Timeline views. The most-used page in the app.
Page Hero
Tasks Hero
Component
Title "Tasks" + stats: total open, overdue (red badge), due today. Right side: view switcher (Board / Table / Calendar / Timeline as segmented control), Filter button, Sort button, Group By button, "+ New Task" primary button. Filters/sort persist between view switches.
Kanban Board View
4 Columns
Component
Columns: To Do (grey dot) / In Progress (blue dot) / Review (amber dot) / Done (green dot, cards muted). Column header: colored dot + name + count badge. Each column has a scroll area (max-height fills screen, overflow-y scroll). "+" icon in column header to add task directly to that column.
Task Card
Interactive
Title (2-line clamp) + priority badge + department badge + due date (red if overdue, accent if today) + assignee avatar + name. Hover reveals: drag handle (6-dot icon) on left, "..." actions menu on right (Edit, Reassign, Change Priority, Delete). Clicking card (not handle) opens the Task Detail Side Panel. Dragging by handle moves card between columns with visual feedback.
hover: lift + border accent glowdragging: rotate(-1deg) scale(1.02) deep shadowdrop zone: dashed border accent color
Drag & Drop (dnd-kit)
Animation
Library: dnd-kit (@dnd-kit/core, @dnd-kit/sortable). Cards are sortable within columns AND draggable between columns. While dragging: original position shows a ghost placeholder (dashed outline). Drop zones glow accent color when a card is hovering over them. On successful drop: card snaps into position, status updates via optimistic UI + API call. On cancel/failed drop: card animates back to origin.
Empty Column State
State
Dashed border zone with text "No tasks here" + "drag cards here" hint + "+ Add task" text link. The drop zone is always visible (taller when empty) to make it easy to drag into.
Table View
Sortable Table
Component
Full-width table. Columns: checkbox, Title (sortable), Status (dropdown), Priority (sortable), Assignee (avatar + name), Due Date (sortable, red if overdue), Department, Project. Column headers are clickable to sort (asc/desc, shows arrow). Hover row shows light background. Clicking Title cell opens Task Detail Panel. Clicking Status/Priority shows an inline dropdown to change value. Bottom row: "+ New row" to add task.
Bulk Selection
Interactive
Checking the header checkbox selects all visible rows. A bulk action bar slides up from the bottom: "X selected · Change Status · Reassign · Delete". Escape or clicking outside deselects. Also accessible via long-press on mobile.
Calendar View
Month Calendar
Component
Standard month grid. Each day cell shows: small date number + up to 3 task chips (priority color dot + truncated title). "+ X more" link if more than 3. Overdue past days show red background tint. Today's cell has accent border ring. Clicking a task chip opens the detail panel. Clicking empty day area opens New Task form pre-filled with that date. Navigation: prev/next month arrows + "Today" button.

400px wide slide-over from the right. Does NOT close the board behind it — board is still visible and interactive. Panel sections: Title (editable inline), Status dropdown, Priority dropdown, Assignee picker (avatar grid), Due date picker, Department, Project link, Description (rich text, Markdown-rendered). Below the fold: Activity log (timeline of all changes + comments). Bottom: Comment input box with submit button. Header: "Open in full page →" link + X close button.

Filters & Search
Filter Panel
Interactive
Clicking "Filter" opens a dropdown panel below the button. Filter options: Status (multi-select checkboxes), Priority (multi-select), Assignee (avatar chips), Department (multi-select), Due Date (presets: Overdue, Today, This Week, This Month + custom date range), Source (Manual, AI, Cadence). ★ My Tasks Must Have
Personal task view filtered to tasks assigned to Saksham, plus self-created tasks (isSelfTask=true). Prioritised list, not kanban. Meant for daily personal planning.
Grouped List
Component
Tasks grouped by: Overdue (red header + count) → Due Today (accent header) → Due This Week → Due Later → No Due Date. Within each group: sorted by priority. Each row is a compact task chip with checkbox (checking = mark done, strikethrough animation), title, priority badge, due date. Dragging rows reorders within a group. Completed tasks collapse to a "X completed today" row, expandable.
Quick Add
Interactive
Sticky "+ Add task" bar at the bottom of each group. Clicking opens an inline form: title input (auto-focus) + priority picker (keyboard shortcut: P) + due date (keyboard: D) + press Enter to save. Escape cancels.

🔄 Cadence Must Have
Recurring meeting schedule: weekly syncs, monthly reviews, 1:1s. Each cadence has prep items that auto-generate tasks N days before. The heartbeat of the team's operating rhythm.
Cadence Cards
Component
Grid of cadence cards. Each card: name + type icon (recurring arrows) + day + time + duration badge + scope (All team / Department) + active/inactive toggle. Expand button to show PrepItems list. Component

📱 Mobile Spec Must Have
The app is currently broken on mobile (fixed 220px sidebar occupies most of a 390px screen). This is the complete mobile redesign spec.
Layout Rules
Breakpoints: <640px = mobile, 640–1024px = tablet, >1024px = desktop.

Mobile: No sidebar. Bottom nav bar (5 tabs). Full-width content. Stacked layouts everywhere. Min touch target 48px. Horizontal scroll for tables (cells fixed-width, table scrolls within container).

Tablet: Collapsible sidebar (hamburger in topbar toggles it). Sidebar overlays content when open (drawer style). Content is full-width when sidebar closed.

Desktop: Fixed 240px sidebar always visible. Topbar with breadcrumb and actions.
