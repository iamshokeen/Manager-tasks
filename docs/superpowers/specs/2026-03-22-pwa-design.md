# PWA — Lohono Command Center
**Date:** 2026-03-22
**Status:** Approved

## Goal
Make the app installable as a standalone app on Android, iOS, and Windows, with offline read support using cached API data.

## Approach
PWA via `@ducanh2912/next-pwa` v2 (actively maintained Workbox fork, supports Next 14–16 and App Router). Service worker generated at build time, served from Vercel.

## `next.config.ts` — exact pattern

`@ducanh2912/next-pwa` v2 default export is a function that takes PWA options and returns a Next.js config wrapper:

```ts
import withPWA from '@ducanh2912/next-pwa'

const nextConfig = { /* existing config */ }

export default withPWA({
  dest: 'public',
  disable: process.env.NODE_ENV === 'development',
  fallbacks: { document: '/offline' },
  workboxOptions: {
    runtimeCaching: [
      {
        urlPattern: /^\/api\/(metrics|targets|numbers)/,
        handler: 'StaleWhileRevalidate',
        options: {
          cacheName: 'api-cache',
          cacheableResponse: { statuses: [0, 200] },
        },
      },
    ],
  },
})(nextConfig)
```

## Caching Strategy

| Route | Strategy | Offline behaviour |
|-------|----------|------------------|
| Static assets (JS/CSS/fonts) | Precache (install-time) | Always available |
| `/api/metrics` | StaleWhileRevalidate | Shows last-fetched data |
| `/api/targets` | StaleWhileRevalidate | Shows last-fetched data |
| `/api/numbers` | StaleWhileRevalidate | Shows last-fetched data |
| Page HTML | NetworkFirst → fallback `/offline` | Shows offline page for uncached routes |
| POST/PATCH routes | NetworkOnly | No offline writes |

## API Route Cache Headers
Add to the `GET` handler response only (not POST/PATCH):
```ts
headers: { 'Cache-Control': 'public, max-age=0, stale-while-revalidate=3600' }
```
This overrides Next.js 15+'s default `no-store` and allows the service worker to cache the response.

## `vercel.json` — SW cache header
```json
{
  "headers": [
    {
      "source": "/sw.js",
      "headers": [{ "key": "Cache-Control", "value": "no-cache, no-store, must-revalidate" }]
    }
  ]
}
```
Prevents Vercel's CDN from caching the service worker file, ensuring users always get the latest SW on update.

## Icons — `scripts/generate-icons.js`
Install once: `npm install --save-dev sharp`

Script content:
```js
const sharp = require('sharp')
const fs = require('fs')

fs.mkdirSync('public/icons', { recursive: true })

const sizes = [192, 512]
for (const size of sizes) {
  const svg = `
    <svg xmlns="http://www.w3.org/2000/svg" width="${size}" height="${size}">
      <rect width="${size}" height="${size}" fill="#C9A84C"/>
      <text
        x="50%" y="55%"
        dominant-baseline="middle"
        text-anchor="middle"
        font-family="sans-serif"
        font-weight="bold"
        font-size="${Math.round(size * 0.55)}"
        fill="#0A0B0F"
      >L</text>
    </svg>`
  sharp(Buffer.from(svg))
    .png()
    .toFile(`public/icons/icon-${size}.png`, (err) => {
      if (err) throw err
      console.log(`Generated icon-${size}.png`)
    })
}
```

Run once with `node scripts/generate-icons.js`, then commit the generated PNGs. Add to `package.json` scripts:
```json
"generate-icons": "node scripts/generate-icons.js"
```

## Install Experience
- **Android/Chrome**: Add to Home Screen prompt after meeting engagement criteria
- **iOS/Safari**: Share → Add to Home Screen
- **Windows/Edge or Chrome**: Install icon in address bar → runs in standalone window, appears in Start menu

## Out of Scope
- Background sync for offline writes
- Push notifications
- Native shell (Capacitor)
