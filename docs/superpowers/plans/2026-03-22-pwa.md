# PWA Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make Lohono Command Center installable as a standalone app on Android, iOS, and Windows with offline read support for the three data API routes.

**Architecture:** `@ducanh2912/next-pwa` (latest, v10.x) wraps `next.config.ts` and generates a Workbox service worker at build time. Static assets are precached; three API routes use StaleWhileRevalidate so offline users see last-synced data. A `/offline` fallback page is served for uncached navigations. Icons generated once with `sharp` (already installed via Next.js) and committed.

**Tech Stack:** `@ducanh2912/next-pwa` v10, Workbox (bundled), `sharp` (already in node_modules via Next.js), Next.js 16, Vercel

## Task 3: Generate app icons

**Files:**
- Create: `scripts/generate-icons.js`
- Create: `public/icons/icon-192.png`
- Create: `public/icons/icon-512.png`
- Create: `public/icons/apple-touch-icon.png`
- Modify: `package.json` (add script entry)

- [ ] **Step 1: Create `scripts/generate-icons.js`**

```js
const sharp = require('sharp')
const fs = require('fs')

fs.mkdirSync('public/icons', { recursive: true })

function makeSvg(size) {
  return `<svg xmlns="http://www.w3.org/2000/svg" width="${size}" height="${size}">
    <rect width="${size}" height="${size}" fill="#C9A84C"/>
    <text
      x="50%" y="55%"
      dominant-baseline="middle"
      text-anchor="middle"
      font-family="sans-serif"
      font-weight="bold"
      font-size="${Math.round(size * 0.55)}"
      fill="#0A0B0F"
    >L</text>
  </svg>`
}

const icons = [
  { size: 192, name: 'icon-192.png' },
  { size: 512, name: 'icon-512.png' },
  { size: 180, name: 'apple-touch-icon.png' },
]

icons.forEach(({ size, name }) => {
  sharp(Buffer.from(makeSvg(size)))
    .png()
    .toFile(`public/icons/${name}`, (err) => {
      if (err) throw err
      console.log(`Generated public/icons/${name}`)
    })
})
```

- [ ] **Step 2: Add `generate-icons` script to `package.json`**

In the `"scripts"` block, add:
```json
"generate-icons": "node scripts/generate-icons.js"
```

- [ ] **Step 3: Run the script**


Expected:

- Modify: `src/app/api/targets/route.ts`
- Modify: `src/app/api/numbers/route.ts` (GET only — file also has POST)

Next.js 15+ sets `Cache-Control: no-store` by default on dynamic route handlers. This header overrides that so Workbox can cache the response.

## Task 8: Configure `next.config.ts` with PWA wrapper

**Files:**

`@ducanh2912/next-pwa` v10 uses a curried call pattern: `withPWA(pwaOptions)(nextConfig)`.

- [ ] **Step 1: Replace `next.config.ts` contents**

```ts
import type { NextConfig } from 'next'
import withPWA from '@ducanh2912/next-pwa'

const nextConfig: NextConfig = {
  /* config options here */
}

export default withPWA({
  dest: 'public',
  disable: process.env.NODE_ENV === 'development',
  fallbacks: { document: '/offline' },
  workboxOptions: {
    runtimeCaching: [
      {
        urlPattern: /^\/api\/(metrics|targets|numbers)/,
        handler: 'StaleWhileRevalidate',
        options: {
          cacheName: 'api-cache',
          cacheableResponse: { statuses: [0, 200] },
        },
      },
    ],
  },
})(nextConfig)
```
